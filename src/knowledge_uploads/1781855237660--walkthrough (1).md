# Walkthrough — Внедрение RBAC и защита маршрутовВ рамках данной итерации была реализована проверка ролей (Role-Based Access Control) на уровне Express-маршрутов, подключен middleware авторизации ко всем защищенным роутам и вынесены секреты токенов из исходного кода.\---## Что было сделано### 1\. Безопасность и Конфигурация* **Переменные окружения**: В файл [.env](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/.env) вынесены секреты:  * `ACCESS\\\\\\\_TOKEN\\\\\\\_SECRET`  * `REFRESH\\\\\\\_TOKEN\\\\\\\_SECRET`* **Конфигурация токенов**: В [token.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/config/token.js) заменено жесткое прописание секретных ключей на чтение из `process.env`.### 2\. Добавление роли в JWT* **Слой сервисов**: В [userService.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/services/userService.js) изменен механизм генерации JWT-токенов. Поле `role` добавлено в payload токена при генерации (`generateAccessToken`, `generateRefreshToken`), а также при вызове в методе `login`. Благодаря этому middleware авторизации может мгновенно считывать роль пользователя без лишних обращений к базе данных.### 3\. Middleware проверки ролей (RBAC)* **Новый файл**: Создан [role.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/middleware/role.js).  * Экспортирует функцию `checkRole(allowedRoles, options)`.  * Проверяет, входит ли роль пользователя из JWT в список разрешенных.  * Поддерживает флаг `allowSelf: true` для самообслуживания (позволяет пользователю получить доступ к своему собственному профилю, даже если у него нет явного права на чтение всех профилей).### 4\. Защита маршрутов* **Маршруты Компаний**: В [companyRoutes.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/routes/companyRoutes.js) добавлены `auth` и `role` middleware:  * Чтение доступно для `admin`, `manager`, `viewer`.  * Создание и редактирование доступно для `admin`, `manager`.  * Удаление доступно только для `admin`.* **Маршруты Контактов**: В [contactRoutes.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/routes/contactRoutes.js) добавлены аналогичные ограничения по ролям.* **Маршруты Сделок**: В [dealsRoutes.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/routes/dealsRoutes.js) интегрированы проверки ролей по аналогии с компаниями и контактами.* **Маршруты Пользователей**: В [userRoutes.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/routes/userRoutes.js) настроена тонкая авторизация:  * Получение списка всех пользователей — доступно для `admin` и `manager`.  * Просмотр профиля — доступно `admin`, `manager`, либо владельцу профиля.  * Редактирование профиля — доступно `admin` либо владельцу профиля.  * Удаление пользователя — доступно только для `admin`.* **Дополнительная защита (Контроллер)**: В [userController.js](file:///c:/Users/aldia/OneDrive/Desktop/miniCRM%20—%20копия/src/controllers/userController.js) добавлена валидация при обновлении: изменять поле `role` (менять права другим или повышать их себе) разрешено исключительно пользователям с ролью `admin`.\---## Проверка работоспособности1. **Запуск**: Успешно выполнена проверка запуска приложения с помощью `node server.js`.2. **Результат**: Лог подтверждает успешную инициализацию сервера:```text   Сервер запущен и работает: http://localhost:3000   ```3.
